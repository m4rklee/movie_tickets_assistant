## 1. 问题

经常在中国电影资料馆观影，购票与取票会产生**电子票截图**和**纸质票**。信息分散在相册与口袋中，无法结构化统计「看了什么、花了多少、常去哪个厅」，也无法支撑后续问答与报表。

## 2. 目标用户

- **主用户**：本人（个人专属票夹，非多租户 SaaS）
- **场景**：资料馆电子票 + 馆内纸质票

## 3. 用户目标（JTBD）

见 [pain-points-and-jtbd.md](./pain-points-and-jtbd.md)。

## 4. MVP 范围

### 输入

- 票种：`electronic`（电子票截图）或 `paper`（纸质票拍照）
- 图片：JPG/PNG，单张

### 处理流程

1. **ParseTicket**：视觉 LLM 提取结构化 JSON → 校验 → 写入草稿表
2. 用户核对字段，可修正
3. **ConfirmTicket**：确认后写入 `movie_tickets`

### 输出

- 识别结果 Markdown 表格 + `draft_id`
- 确认后返回 `ticket_id` 与入库摘要

### 电子票字段

电影名称、观看日期、影片时间、观看影院、观看影厅、支付状态、票面价格

### 纸质票字段

电影名称、观看日期、观看影院地址、观看影厅、座位、票价

### 非目标（Out of Scope）

- 猫眼/淘票票等通用影院模板（v1）
- 飞书 / Langbot 渠道（v2）
- 票夹问答、观看报表（v3）
- 多用户账号、支付、退票、转赠

## 5. 成功指标

| 指标 | MVP 目标 |
|------|----------|
| 核心字段准确率（片名、日期、影厅、价格） | ≥ 95%（10 张金标准集） |
| 电子票：时间、支付状态 | ≥ 90% |
| 纸质票：座位 | ≥ 90% |
| 端到端入库耗时（含确认） | ≤ 2 分钟/张 |
| 幻觉（编造场次/价格） | 0 次（内测） |

## 6. 方案概要

- **形态**：Dify **Workflow** × 2（Parse + Confirm）+ **ticket-api**（FastAPI + MySQL）
- **差异化**：资料馆双票种 schema + 人工确认 + 字段级评测集
- **参考**：[dify-for-dsl](https://github.com/wwwzhouhui/dify-for-dsl) 票据识别 DSL；MatchCV 评测方法论
